## X. CSS Background

### 1. Background color

The CSS **background-color** property allows you to set the background color of an element.

```css
body {
  background-color: lightblue;
}
```

This will set the background color of the **body** element to light blue.

You can also specify the background color using a hexadecimal value:

```css
body {
  background-color: #add8e6;
}
```

This will also set the background color of the **body** element to light blue.

In addition to color names and hexadecimal values, you can also specify the background color using the RGB or RGBA color models:

```css
body {
  background-color: rgb(173, 216, 230);
}

body {
  background-color: rgba(173, 216, 230, 0.5);
}
```

The first example sets the background color of the **body** element to light blue using the RGB color model. The second example sets the background color using the RGBA color model, with an alpha value of 0.5 to make the background semi-transparent.

### 2. Background image

A **background image** is an image that is placed behind the content of a webpage. To add a background image in CSS, you can use the **background-image** property along with the **background-size** and **background-repeat** properties.

- Simple background image:

```css
body {
  background-image: url(image.jpg);
  background-size: cover;
  background-repeat: no-repeat;
}
```

- Fixed background image:

```css
body {
  background-image: url(image.jpg);
  background-size: cover;
  background-repeat: no-repeat;
  background-attachment: fixed;
}
```

- Background image with background color:

```css
body {
  background-image: url(image.jpg);
  background-size: cover;
  background-repeat: no-repeat;
  background-color: #333;
}
```

- Centered background image:

```css
body {
  background-image: url(image.jpg);
  background-size: cover;
  background-repeat: no-repeat;
  background-position: center;
}
```

## XI. CSS Transitions

The CSS **transition** property allows you to specify the transition effect when changing from one style to another. It makes an element smoothly change from one state to another, instead of an abrupt change.

- **Transition-property**: defines the CSS property that should be animated during a transition. For example, you can animate the color, width, height, or any other CSS property.
- **Transition-duration**: sets the duration of the transition effect. The value is specified in seconds or milliseconds.
- **Transition-timing-function**: defines the speed curve of the transition. This property determines how the animation progresses over time. Some common values include **linear**, **ease**, **ease-in**, **ease-out**, **ease-in-out**, and **cubic-bezier**.
- **Transition-delay**: sets a delay before the transition effect begins. The value is specified in seconds or milliseconds.

```css
button {
  transition-property: background-color;
  transition-duration: 0.5s;
  transition-timing-function: ease-in-out;
  transition-delay: 0.2s;
}

button:hover {
  background-color: lightblue;
}
```

In this example, the transition effect is applied to the background-color property of the **button** element. When the mouse hovers over the button, the background color will smoothly change to light blue over a duration of 0.5 seconds, with an ease-in-out timing function and a delay of 0.2 seconds before the transition starts.

You can also combine multiple transition properties into a single shorthand property:

```css
button {
  transition: background-color 0.5s ease-in-out 0.2s;
}
```

## XII. CSS Layout

**CSS layout** refers to the process of arranging and managing the positions and sizes of elements within a web page using CSS (Cascading Style Sheets). It allows for precise control over the layout of HTML elements on a page, including things like spacing, positioning, and dimensions.

There are several CSS layout methods, including:

- Normal Flow
- Floats (Deprecated)
- Positioning
- Flexbox
- Grid Layout

### 1. Normal Flow

**Normal flow** in CSS refers to the default layout behavior of HTML elements on a web page. In normal flow, elements are displayed one after another in the order they appear in the HTML document, and the size of an element is determined by its content.

In normal flow, block-level elements, such as **div** and **p**, are stacked vertically and take up the full width of their parent container. Inline elements, such as **span** and **a**, only take up as much width as their content requires and can appear within a block-level element.

The **display** property in CSS is used to define the type of layout model that an HTML element should use. The value of the **display** property determines how an element is displayed within the document and affects its behavior, size, and position.

- **block**: A block-level element takes up the full width of its parent container and creates a new block formatting context.

```css
div {
  display: block;
}
```

- **inline**: An **inline** element takes up only as much width as its content requires and does not create a new block formatting context.

```css
span {
  display: inline;
}
```

- **inline-block**: An **inline-block** element takes up only as much width as its content requires and allows for setting dimensions, such as width and height, and margin and padding.

```css
span {
  display: inline-block;
  width: 50px;
}
```

- **none**: An element with **display: none** will not be visible on the page and will take up no space.

```css
p {
  display: none;
}
```

### 2. Positioning

The CSS **position** property is used to specify the type of positioning method used for an element. There are four possible values for the **position** property: **static**, **relative**, **fixed**, and **absolute**.

- **Static**: Elements are positioned according to the normal flow of the document and can't be positioned using top, bottom, left, and right properties. (default value)

```css
.static {
  position: static;
}
```

- **Relative**: Elements are positioned relative to their normal position, and you can use top, bottom, left, and right properties to adjust the element's position.

```css
.relative {
  position: relative;
  top: 20px;
  left: 30px;
}
```

- **Absolute**: Elements are positioned relative to the nearest positioned ancestor (instead of positioned relative to the viewport, like fixed), but without leaving a gap in the normal flow of the document.

```css
.absolute {
  position: absolute;
  top: 40px;
  right: 50px;
}
```

- **Fixed**: Elements are positioned relative to the viewport, and they will not move even if the page is scrolled.

```css
.fixed {
  position: fixed;
  bottom: 10px;
  right: 10px;
}
```

### 3. Flexbox

**Flexbox** is a CSS layout module that makes it easier to create flexible and responsive layouts.
![Flexbox CSS](https://images.viblo.asia/0c116fbb-972d-4162-8506-bac8605cd704.png)

- Creating a flex container:

```css
.container {
  display: flex;
}
```

- Setting the direction of flex items:

```css
.container {
  display: flex;
  flex-direction: row; /* default value */
}

/* To change the direction to column */
.container {
  display: flex;
  flex-direction: column;
}
```

- Aligning flex items along the cross axis:

```css
.container {
  display: flex;
  align-items: center; /* default value is stretch */
}
```

- Justify flex items along the main axis:

```css
.container {
  display: flex;
  justify-content: space-between; /* default value is flex-start */
}
```

- Wrapping flex items:

```css
.container {
  display: flex;
  flex-wrap: wrap; /* default value is nowrap */
}
```

- Setting the order of flex items:

```css
.item-1 {
  order: 1;
}

.item-2 {
  order: 2;
}
```

### 4. Grid Layout

**CSS Grid Layout** is a two-dimensional layout system that lets you create complex and flexible grid-based layouts.
![Grid CSS](https://www.w3schools.com/css/grid_lines.png)

- Creating a grid container:

```css
.container {
  display: grid;
}
```

- Defining the number of columns and rows:

```css
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 100px 100px;
}
```

- Placing items on the grid:

```css
.item-1 {
  grid-column: 1 / 2;
  grid-row: 1 / 2;
}

.item-2 {
  grid-column: 2 / 3;
  grid-row: 1 / 2;
}

.item-3 {
  grid-column: 3 / 4;
  grid-row: 1 / 2;
}
```

- Using grid areas:

```css
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 100px 100px;
  grid-template-areas:
    "header header header"
    "sidebar main main";
}

.header {
  grid-area: header;
}

.sidebar {
  grid-area: sidebar;
}

.main {
  grid-area: main;
}
```

## XIII. CSS Animation

**CSS Animations** allows you to apply animations to elements in your HTML documents. Animations are a series of changes to CSS styles over a set period of time.

- **@keyframes**: used to define the different stages of an animation. The **@keyframes** rule specifies the CSS styles for each keyframe.
- **animation-name**: used to specify the name of the **@keyframes** animation.
- **animation-duration**: used to specify the duration of the animation. The duration is specified in seconds or milliseconds.
- **animation-timing-function**: used to specify the speed curve of the animation. It controls the acceleration and deceleration of the animation.
- **animation-delay**: used to specify the delay before the animation starts. The delay is specified in seconds or milliseconds.
- **animation-iteration-count**: used to specify the number of times an animation should run.
- **animation-direction**: used to specify the direction of the animation. The animation can run forwards, backwards, or alternate between the two.
- **animation-fill-mode**: used to specify the state of the element when the animation is not playing.
- **animation-play-state**:used to specify whether an animation is playing or paused.

```css
div {
  width: 100px;
  height: 100px;
  background-color: red;
  animation-name: example;
  animation-duration: 4s;
  animation-timing-function: linear;
  animation-delay: 2s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
  animation-fill-mode: forwards;
  animation-play-state: running;
}

@keyframes example {
  0% {
    transform: translate(0, 0);
  }
  100% {
    transform: translate(200px, 200px);
  }
}
```

You can also combine multiple animation properties into a single shorthand property:

```css
div {
  width: 100px;
  height: 100px;
  background-color: red;
  animation: example 4s linear 2s infinite alternate forwards running;
}
```

## XIV. Media Queries

**CSS Media Queries** allow you to apply different styles to your HTML documents based on the conditions of the device displaying the content. This can be based on the type of device, such as a screen or a printer, or on specific features of the device, such as its width or height.

### 1. Media types

- **Screen**: The **screen** media type applies to devices with a screen, such as a computer, tablet, or smartphone.
- **Print**: The **print** media type applies to devices used for printing, such as printers or print preview in a web browser.
- **Speech**: The **speech** media type applies to speech synthesizers.

### 2. Media features

- **Width**: The **width** media feature allows you to apply styles based on the width of the viewport of the device displaying the content.
- **Height**: The **height** media feature allows you to apply styles based on the **height** of the viewport of the device displaying the content.
- **Orientation**: The **orientation** media feature allows you to apply styles based on the orientation of the device displaying the content. The **orientation** can be either portrait or landscape.
- **Resolution**: The **resolution** media feature allows you to apply styles based on the resolution of the device displaying the content. The **resolution** is specified in dots per inch (dpi) or dots per centimeter (dpcm).

```css
@media only screen and (min-width: 600px) {
  body {
    background-color: lightblue;
  }
}

@media print {
  body {
    font-size: 15px;
  }
}

@media (orientation: portrait) {
  body {
    background-color: pink;
  }
}
```

## XV. CSS Preprocessor

**CSS Preprocessors** are scripting languages that extend the capabilities of CSS and make it easier to write and maintain CSS code. CSS preprocessors allow you to use **variables**, **mixins**, **functions**, and other features that are not available in regular CSS.

### 1. Sass (Syntactically Awesome Style Sheets)

- **Variables**: Sass allows you to define variables in your stylesheets, making it easier to maintain and update your code.

```css
$primary-color: #333;
$font-stack: Helvetica, sans-serif;
$width: 100px;

body {
  background-color: $primary-color;
  font-family: $font-stack;
  width: $width;
}
```

- **Mixins**: Sass mixins allow you to reuse CSS styles, making it easier to maintain your code and avoid repetitive code.

```scss
@mixin border-radius($radius) {
  -webkit-border-radius: $radius;
  -moz-border-radius: $radius;
  -ms-border-radius: $radius;
  border-radius: $radius;
}

.box {
  @include border-radius(10px);
}
```

- **Nesting**: Sass allows you to nest CSS styles, making your code easier to read and maintain.

```scss
nav {
  ul {
    margin: 0;
    padding: 0;
    list-style: none;
  }

  li {
    display: inline-block;
    background-color: #333;
    a {
      display: block;
      padding: 10px;
      color: #fff;
      text-decoration: none;
    }
  }
}
```

- **Partials**: Sass partials allow you to break your stylesheets into smaller, reusable chunks, making it easier to manage and maintain your code.

```scss
/* _header.scss */
header {
  background-color: #333;
  height: 80px;
  color: #fff;
}
```

```scss
/* style.scss */
@import "header";
```
